# Section catalog (Layer 3)

The landing-page section library. Stage 02 picks an ordered subset for the spec;
stage 03 designs each. Every chosen section must serve the page's primary objective.

## Core sections

| Section | Purpose | Include when | Key content |
|---------|---------|--------------|-------------|
| **Hero** | State value + drive primary CTA | Always | Headline (outcome), subhead (who/how), primary CTA, supporting visual |
| **Social proof bar** | Borrow trust early | Has logos/press/ratings | Customer/press logos, rating, or a one-line stat |
| **Problem / pain** | Name the status quo that hurts | Pain is the wedge | Plain statement of the problem the visitor feels |
| **Solution / value** | Show how you close the gap | Always (may merge w/ features) | The core mechanism, framed as outcome |
| **Features / benefits** | Explain what they get | Multiple capabilities to convey | 3–6 benefits, each outcome-led, with an icon |
| **How it works** | Reduce uncertainty | Process isn't obvious | 3–4 numbered steps |
| **Testimonials** | Credibility via real voices | Have quotes/case studies | Quote, name, role, company, photo/logo |
| **Stats / impact** | Make impact concrete | Have real numbers | Big number + short label blocks |
| **Pricing** | Remove cost ambiguity | Self-serve / transparent pricing | Tiers, what's included, CTA per tier |
| **FAQ** | Handle objections | Recurring questions/objections | 4–8 Q&A, the real blockers |
| **Lead capture / CTA band** | Convert | Always (closing CTA) | Restated value + primary CTA (+ short form if applicable) |
| **Footer** | Navigation, legal, contact | Always | Links, contact, social, legal, copyright |

## Section patterns by business type

A sensible default arc per type — adapt to the specific business and goal.

- **SaaS / software:** Hero → Social proof → Problem → Solution → Features → How it
  works → Testimonials → Pricing → FAQ → CTA band → Footer.
- **Local services** (trades, clinics, salons): Hero → Trust (reviews/credentials) →
  Services → How it works → Gallery/Before-after → Testimonials → Service area/Hours
  → Contact/Book → Footer.
- **E-commerce / product:** Hero → Social proof → Benefits → Product highlights →
  How it works/Use → Reviews → Guarantee/Shipping → FAQ → CTA → Footer.
- **Professional services** (law, accounting, consulting): Hero → Credibility →
  Services/Practice areas → Approach → Case results/Testimonials → Team → FAQ →
  Contact/Consultation → Footer.
- **Agency / studio:** Hero → Client logos → Services → Selected work → Process →
  Testimonials → Team → Contact → Footer.
- **Generic / other:** Hero → Value → Features/Benefits → Proof → How it works → FAQ
  → CTA → Footer.

## Rules

- **Lead with the hero**, end with a **closing CTA + footer**. Everything between
  builds the case.
- **No section without a job.** If you can't tie it to the objective, drop it.
- **Place proof near decisions** — right after the hero and right before CTAs.
- **One primary CTA**, repeated; secondary CTAs stay visually subordinate.
- Keep the page **as short as it can be while making the case** — every section is a
  reason to scroll, not a reason to leave.
